# SPA-BO/DNN 控制台 - 项目状态说明 (Project Status)

本文件说明了当前“数字孪生沙盒控制台”已经实现的功能结构，以及未来有待接入和完善的系统缺陷与待开发计划。

## 🟢 已实现的功能 (Implemented Features)

### 1. 数字孪生 UI 架构与交互界面 (Digital Twin Interface)
* **实时全局响应式布局**：修复了视口拉长溢出问题，目前三栏式面板采用 `h-screen` 全屏无滚动设计，内部卡片实现了完整的自适应滚动与占位。
* **中英双语标签支持**：对控制台所有的英文参数和模块标题（如 Sensors、Hardware、UCB Horizon 等）添加了易懂的中文释义与状态标明。

### 2. 环境传感矩阵监控 (Sensor Monitoring Matrix)
* 模拟了四大核心实时指标面板：
  * **蓝藻密度 (Algae)**
  * **酸碱度 (pH)**
  * **水体温度 (Temp)**
  * **反应电导率 (Cond)**
* 对数据做出了动态波动效果，并将历史数据转化为底层趋势折线图直观显示。

### 3. 可视化寻优观测核心 (Visualization & Optimization Hub)
* **KPI 计量卡片**：直观展示当前的**灭活率 (Inactivation Rate)**、**能耗 (Energy Consumed/Power)** 和 **BO目标得分 (BO Target Score)**。
* **态势演化矩阵 (Evaluation Topology)**：引入了 `recharts` 提供了复合图表，能够动态观察双Y轴下 Inact% 与 Power 相互制衡优化的曲线。
* **代理网络与候选空间**：实时绘制模拟的 **DNN 网络损失曲线 (Loss)** 和 **UCB 置信界候选解散点图 (Candidates)**，反映算法幕后状态。

### 4. 硬件链路与 AI 自动化接管 (Hardware Control & AI Override)
* **基础硬件控制参数面板**：提供对**球隙激发距离**、**钨针辐射间距**和**电容脉冲能级**的手动滑块（Slider）实时调节。
* **自动/手动模式切换保护**：
  * 点击“启用AI全自动闭环寻优”后，底层算法接管操作。
  * 手动硬件控制滑块进入半透明不可交互状态（禁用拖拽），UI 弹出锁定提示屏蔽人为干扰。

### 5. 系统控制台日志 (System Logs/Terminal)
* 搭建了自动滚动的日志终端墙，实时记录了实验数据迭代、模型寻优信息、操作模式变更等系统日志。

## 🔴 未实现 / 待开发的功能 (To-Be-Implemented / Future Features)

目前页面主要为一个纯前端构建的**沙盒演示环境（Mock/Sandbox）**，下列功能仍需要后续开发：

### 1. 真实物理后端环境接入 (Real Backend & WebSocket Integration)
* **现状**：传感器数据和图表点位是由前端 `setInterval` 和数学算法模拟生成的。
* **待办**：需要开发真实设备的后端服务，使用 WebSocket 或 MQTT 协议将真实的工业传感器数据推送至前端。

### 2. 真实 DNN 与贝叶斯优化算法打通 (True Algorithm Integration)
* **现状**：**前端已成功完整集成了真实的深度学习网络！** 控制台内部使用 `@tensorflow/tfjs` 构建了真实的 DNN 代理模型（Surrogate Model），并在浏览器端实时执行真实的张量运算（Tensor Operations）、反向传播梯度下降（Adam / MSE）与模型拟合（Fit）。结合前端逻辑，实现了一套“闭环的轻量级贝叶斯寻优（BO）系统”。目前展现的 Loss 曲线是真实的训练计算结果，而不是纯随机数生成的假动画。
* **待办（进阶）**：虽然浏览器端的 TF.js 足以演示和处理中小型问题，但在工业级部署时，为了减轻网页浏览器的计算负担和处理海量历史数据池，通常会在此处**剥离前端计算**，将核心调度和模型推理由浏览器转移到后端的 Python 专属服务器（PyTorch/TensorFlow）节点进行。

### 3. 持久化存储与历史数据导出 (Database & Data Export)
* **现状**：刷新浏览器页面后，所有累计的迭代数据、图表和系统日志都会被重置清空。
* **待办**：引入数据库（如 Firebase, Supabase 或 SQL）。增加“保存实验批次 (Experiment Batch)”、以及**导出当前实验数据为 CSV/Excel** 的功能。

### 4. 身份验证与权限系统 (Authentication & RBAC)
* **现状**：任何人获得网址即可进入并点击自动化按钮。
* **待办**：引入登录系统及权限控制（RBAC），划定【观察者模式】（仅可查看数据）和【操作员模式】（可更改硬件参数和启动AI），并要求关键操作具有二次确认流程保护设备安全。

### 5. 危险阈值与异常告警闭环 (Alerts & Emergency Overrides)
* **现状**：当水体温度或酸碱度出现极端情况时，UI没有任何强制停机和越界警告。
* **待办**：设置物理和生物极值告警（如温度过高自动停机机制），并且可通过邮件或短信等形式通知项目负责人。
